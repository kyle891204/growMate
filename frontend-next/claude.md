# GrowMate 프로젝트 정리

## 1. 프로젝트 개요

**프로젝트명:** GrowMate  
**주제:** 사용자와 상호작용하는 반려식물 스마트 화분  
**형태:** 라즈베리파이 기반 스마트 화분 + 웹앱 UI

GrowMate는 단순히 식물을 자동으로 키워주는 스마트화분이 아니라, 사용자가 식물과 감정적으로 교감하면서 직접 돌보는 경험을 유지하도록 돕는 반려식물형 스마트 화분 시스템이다.

기존 스마트화분은 주로 자동급수, 센서 모니터링, 앱 알림에 초점을 둔다. GrowMate는 여기에 **LLM 기반 공감채팅**, **사용자 참여형 돌봄 선택**, **OLED 표정**, **서보모터 잎 모션**, **센서/행동 로그 일지**를 결합한다.

---

## 2. 핵심 컨셉

### 핵심 한 문장

> GrowMate는 식물 상태와 사용자 감정을 함께 반영하여, 식물이 표정·메시지·잎 모션으로 반응하는 양방향 감성 상호작용 스마트 화분이다.

### 핵심 차별점

1. **LLM 기반 공감채팅**
   - 사용자가 하루의 스트레스나 감정을 입력하면 식물이 공감·위로 메시지를 제공한다.
   - 예: “오늘 과제도 많고 너무 지쳤어.” → “오늘 정말 고생 많았어요. 제가 옆에서 들어드릴게요.”

2. **사용자 참여형 돌봄**
   - 물이 부족하거나 조도가 낮을 때 자동으로 바로 처리하지 않고, 먼저 식물이 사용자에게 제안한다.
   - 예: “조금 목말라요. 물을 줄까요?”
   - 사용자는 `[물 주기]`, `[나중에]` 같은 선택을 할 수 있다.
   - 단, 장시간 미응답 시 자동 보호 모드로 식물이 죽지 않게 관리한다.

3. **서보모터 기반 잎 모션**
   - OLED 표정뿐 아니라, 실제 잎 모양 파츠가 움직이며 식물의 반응을 물리적으로 표현한다.
   - 예: 물을 주면 잎이 기쁘게 흔들림, 목마르면 아래로 처짐.

4. **상호작용 로그/식물 일지**
   - 센서 변화, 사용자 선택, 급수, LED 작동, 공감채팅 기록을 일지로 남긴다.

## 3. 기존 제품과의 비교 관점

### 주요 비교 제품

- LeafyPod / 리피팟
- Ivy Smart Planter / 아이비
- Planty

### 주의할 점

기존 제품도 이미 다음 기능을 제공한다.

- 자동급수
- 센서 기반 상태 확인
- 앱 연동
- 표정 애니메이션
- 식물 상태 알림
- 일부 AI 메시지 또는 감성형 인터페이스

따라서 GrowMate의 차별점은 단순히 “표정이 있다” 또는 “AI가 말한다”가 아니다.

### GrowMate의 진짜 차별점

> Planty가 식물에게 AI 목소리와 성격을 부여한다면, GrowMate는 그 대화를 실제 물 주기, 조명 제어, 서보모터 잎 모션으로 연결한다.

즉, GrowMate는 **AI 상호작용 + 실제 피지컬 컴퓨팅 제어**가 결합된 프로젝트다.

## 4. 하드웨어 구성

### 주요 부품

- Raspberry Pi
- 온습도 센서
- 토양 수분 센서
- MCP3008
- 조도 센서
- 미니 수중 워터펌프
- 릴레이 모듈 또는 MOSFET 모듈
- 식물 생장 LED
- 실리콘 호스
- 3.5인치 디스플레이 또는 OLED 디스플레이
- 외부 물통
- 화분
- 화분 받침대
- 서보모터
- 흙
- 식물

### 외형 구조

- 화분 전면: 표정 디스플레이
- 화분 측면: 잎 모양 반응 파츠 1개
- 화분 뒤쪽: 높이 조절형 LED 스탠드 또는 구스넥 LED
- 물통: 화분 내부가 아니라 외부에 배치
- 전자부품: 가능하면 화분 뒤쪽/아래쪽에 정리

### 잎 파츠

- 잎 자체는 포맥스 2T 또는 폼보드로 제작
- 서보모터로 위아래 또는 좌우 움직임 구현
- 3D 프린터는 잎 자체보다 서보모터 브라켓, OLED 프레임, LED 고정부 제작에 활용

### LED

- 밝기 조절은 필수 아님
- 우선순위는 웹에서 ON/OFF 제어, 조도 부족 시 LED 켜기 제안, 자동 보호 모드에서 LED ON
- 완제품 USB 식물 LED를 쓸 경우 웹 제어는 USB 전원 ON/OFF 수준이 현실적
- 12V LED 바 + MOSFET을 쓰면 더 안정적인 제어 가능

# 6. 화면별 기능 명세

## 6.1 홈 화면

### 목적

식물의 현재 상태를 한눈에 보여주고, 식물이 필요한 돌봄을 사용자에게 제안하는 메인 화면이다.

### 핵심 원칙

- 식물이 화면에서 가장 크게 보여야 한다.
- 사용자가 식물을 터치하면 상호작용할 수 있어야 한다.
- 홈 화면에는 무조건적인 “빠른 실행” 버튼 묶음을 두지 않는다.
- 식물이 필요하다고 말할 때만 물 주기나 LED 켜기 버튼이 말풍선 형태로 등장한다.

### 포함 요소

- 상단 제목: `GrowMate`
- 알림 아이콘
- 큰 식물 캐릭터 영역
- 식물 터치 유도 문구
  - 예: `식물을 터치해보세요!`
- 식물 옆 말풍선 제안
  - 예: `조금 목말라요. 물을 줄까요?`
  - 버튼: `물 주기`, `나중에`
- 조도 부족 제안
  - 예: `조금 어두운 것 같아요. 빛이 더 필요할지도 몰라요.`
- 식물 상태 카드
  - 예: `조금 목이 마른 것 같아요`
  - 설명: `흙이 건조해졌어요. 물을 주면 다시 싱그럽게 자랄 거예요!`
- 센서 요약 카드
  - 토양 수분
  - 온도
  - 습도
  - 조도

### 홈 화면에서 하지 말아야 할 것

- 고정된 빠른 실행 버튼 묶음
  - `물 주기`
  - `LED 켜기`
  - `자동 모드`
  - `오늘 상태 보기`
- 공감채팅 기능을 홈에 넣지 않기
- 사용자가 아무 때나 물을 줄 수 있게 만드는 UX 지양

---

## 6.2 공감채팅 화면

### 목적

식물 돌봄 제어가 아니라, 사용자가 하루의 감정과 스트레스를 털어놓고 식물에게 공감과 위로를 받는 화면이다.

### 핵심 원칙

- 물 주기/LED 제어 버튼을 넣지 않는다.
- LLM 기반 감정 공감 대화 화면으로 보이게 한다.
- 사용자의 감정 입력에 식물이 따뜻하게 반응한다.

### 포함 요소

- 제목: `공감채팅`
- 상단 카드
  - 작은 식물 캐릭터
  - `LLM 공감 모드`
  - `오늘 하루 어땠나요?`
  - `언제든 이야기해 주세요. 제가 따뜻하게 들어드릴게요.`
- 채팅 영역
  - 사용자 메시지 오른쪽 정렬
  - 식물 메시지 왼쪽 정렬
  - 식물 메시지 옆에 작은 식물 아이콘
- 예시 대화
  - 사용자: `오늘 과제도 많고 너무 지쳤어.`
  - 식물: `오늘 정말 고생 많았어요. 많이 지쳤겠네요. 잠깐 숨을 고르고, 제가 옆에서 들어드릴게요.`
  - 사용자: `괜히 마음이 불안해.`
  - 식물: `그럴 수 있어요. 지금처럼 솔직하게 말해주는 것만으로도 잘하고 있어요. 오늘 가장 힘들었던 순간이 언제였나요?`
- 추천 프롬프트 칩
  - `오늘 너무 힘들었어`
  - `위로해줘`
  - `기분이 불안해`
  - `오늘 잘한 일 알려줘`
- 입력창
  - placeholder: `오늘의 감정을 이야기해보세요`
- 전송 버튼

## 6.3 일지 화면

### 목적

사용자와 식물의 상호작용 기록, 센서 변화, 급수, LED, 공감채팅 기록을 시간순으로 보여준다.

### 핵심 원칙

- 단순 센서 로그가 아니라 “상호작용형 식물 일지”로 보여야 한다.
- 사용자 행동과 식물 반응을 함께 기록한다.

### 포함 요소

- 제목: `일지`
- 오늘의 기록 요약 카드
  - 식물 이미지/아바타
  - `오늘의 기록`
  - `오늘 물 주기 1회, LED 2회, 상태 안정`
  - `식물이 편안한 하루였어요. 계속 잘 돌봐주고 있어요!`
- 필터 칩
  - `전체`
  - `급수`
  - `LED`
  - `대화`
  - `센서`
- 타임라인 로그
  - `14:21 물 주기 완료`
    - `사용자 직접 선택`
  - `15:00 LED 켜짐`
    - `조도 부족으로 보조 조명 작동`
  - `17:10 기분 대화`
    - `사용자 감정: 지침 / 식물 반응: 위로 메시지`
  - `18:30 상태 안정`
    - `토양 수분 정상, 조도 정상`
- 주간 변화 그래프 카드
  - 제목: `주간 변화`
  - 필터: `최근 7일`
  - 표시 데이터: 토양 수분, 조도

## 6.4 설정 화면

### 목적

식물 프로필, 알림, 돌봄 모드, 공감채팅, 연결 상태를 관리한다.

### 포함 요소

- 제목: `설정`
- 식물 프로필 카드
  - 식물 이미지
  - `내 식물 프로필`
  - 이름: `그로우메이트`
  - 종류/상태: `몬스테라 · 연결 정상`
- 알림 설정
  - `물 주기 알림`
  - `조도 부족 알림`
  - `공감채팅 알림`
- 돌봄 모드
  - `자동 보호 모드`
  - `LED 자동 보조`
  - `제안 메시지 표시`
- 공감채팅 설정
  - `대화 기록 저장`
  - `감정 공감 톤`
  - 값: `따뜻하게`
  - `식물 별명 설정`
- 연결 및 시스템
  - `센서 상태 확인`
  - `Wi-Fi 연결`
  - `장치 정보`
- 하단 버튼
  - `로그아웃`
  - `초기화`

# 8. 프론트엔드 구현 요구사항

## 기술 스택

- **Next.js (App Router, `app/` 디렉터리) + React**
- JavaScript 사용 (TypeScript는 강제하지 않음, 단 일관성 유지)
- 스타일: **CSS Modules 또는 일반 CSS만 사용**
- **외부 UI/스타일 라이브러리 사용하지 않기** (Tailwind, MUI, styled-components, shadcn 등 금지)
- 아이콘·차트도 외부 라이브러리 대신 **인라인 SVG**로 직접 구현 (의존성 최소화)
- 백엔드 연동 없이 **mock data로 구현** (API Route·DB·서버 fetch 없음, 데이터는 `lib/data/`에서 import)
- `create-next-app` 세팅 시 **Tailwind 옵션은 끈다**, App Router 사용
- `npm run dev`로 바로 실행 가능해야 함

## 레이아웃 규칙

- 모바일 화면 폭 약 390px 기준
- 화면 중앙 정렬
- 하단 탭바 고정(fixed), 콘텐츠는 탭바 높이만큼 하단 패딩 확보

## 라우팅 / 탭 구현

- 탭은 **App Router 라우트로 구현**한다 (상태 토글 방식 X)
  - `/` → 홈
  - `/chat` → 공감채팅
  - `/diary` → 일지
  - `/settings` → 설정
- 고정 탭바는 루트 레이아웃(`app/layout.js`)에 두어 모든 화면에 공통 표시
- 화면 이동은 `next/link`의 `<Link>`, 현재 탭 강조는 `usePathname()`으로 활성 경로 판별
- 탭 순서 고정: 홈 / 공감채팅 / 일지 / 설정

## 컴포넌트 구분

- 토글·채팅 입력·필터 등 상호작용이 있는 컴포넌트는 **`"use client"`** 로 작성
- 정적 레이아웃은 서버 컴포넌트로 둬도 됨
- 토글·필터·입력 등 상호작용은 **클라이언트 컴포넌트의 로컬 state로 실제 반응**하게 구현 (서버 호출 없음)

## 권장 폴더 구조

```
app/
  layout.js            # 루트 레이아웃: 모바일 프레임 + 고정 TabBar
  globals.css          # 디자인 토큰 + 전역 스타일
  page.js              # 홈 (/)
  chat/page.js         # 공감채팅 (/chat)
  diary/page.js        # 일지 (/diary)
  settings/page.js     # 설정 (/settings)
components/            # TabBar, Card, Pill, Toggle, SpeechBubble, SensorCard 등
lib/
  data/                # mock 데이터 (plant, sensors, suggestions, chat, logs, settings)
```
